关于「完整网站」与 GitHub Pages

排版智能分析器（Next.js）· 线上说明页

为什么 Pages 上打不开完整功能？

GitHub Pages 只提供静态文件托管，不能运行 Node.js，也没有本项目需要的 /api/analyze、字库上传等后端逻辑。因此不存在一个「在 GitHub Pages 上即可上传图片并完成全流程分析」的链接。

体验完整网站的方式

  • 在本机克隆仓库后进入 typography-grid-detector，执行 npm install 与 npm run dev（详见仓库 README）。
  • 或使用 Vercel / 云服务器 / Docker 部署整套 Next 应用（见仓库内 DEPLOY.md）。部署成功后，你会得到自己的 https://… 链接，那才是「完整网站」的演示地址。
